# CS010. Async function has no asynchronous calls or features
Declaring a function as `async` without using any asynchronous features (`await`, `async with`, `async for`) is 
misleading. It adds unnecessary complexity and signals async behavior where none exists.

### ❌ Anti-pattern
```python
async def get_user(user_id):  # There is no reason to use async
    return users[user_id]
```

### ✅ Best practice
```python
def get_user(user_id):
    return users[user_id]
```
