# CS008. Nested "with" statements of the same type should be comma-separated
Nested `with` statements create unnecessary indentation and reduce readability. Use comma-separated context managers instead.

Why we consider it a best practice:
- Flatter structure → better readability: avoids deep nesting and keeps the main logic easier to scan.
- Less visual noise: reduces indentation and boilerplate.
- Easier modification: adding, removing, or reordering context managers (e.g., mocks) is simpler.

### ❌ Anti-pattern
```python
with method1(arg) as variable1:
    with method2(arg) as variable2:
        with method3(arg) as variable3:
            do_something(...)
```

### ✅ Best practice
```python
with (
    method1(arg) as variable1,
    method2(arg) as variable2,
    method3(arg) as variable3,
):
    do_something(...)
```

### ⚠️ Exception
You cannot mix `with` and `async with` in a single statement. When combining sync and async context managers, 
nesting is required:
```python
with method1(arg) as variable1:
    async with method2(arg) as variable2:
        do_something(...)
```
